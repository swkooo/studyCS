## ❓ 예상 질문

### SQL 조인 및 원리 📊
1.  **INNER JOIN과 LEFT JOIN의 차이점**을 벤 다이어그램과 함께 설명하세요.
2.  대용량 테이블 조인 시 **Hash Join**이 **Nested Loop Join**보다 유리한 이유는 무엇인가요?
3.  **Sort-Merge Join**의 과정과 시간 복잡도($O(N \log N)$)를 설명하세요.

### 복잡도 및 선형 자료 구조 📏
1.  **시간 복잡도**의 개념과 **Big $O$ 표기법**을 사용하는 이유를 설명하세요.
2.  **배열(Array)**과 **연결 리스트(Linked List)**의 삽입/삭제 연산의 시간 복잡도를 비교하고 그 이유를 설명하세요.
3.  **스택(Stack)**과 **큐(Queue)**가 각각 적용되는 실무 사례를 드세요.

### 그래프 🌐
1.  그래프의 표현 방법(인접 행렬, 인접 리스트)의 장단점은?
2.  DFS와 BFS의 차이점과 각각의 사용 사례는?
3.  그래프에서 사이클을 탐지하는 방법은?

### 트리 🌳
1.  이진 트리와 이진 탐색 트리의 차이점은?
2.  트리의 순회 방법(전위, 중위, 후위)을 설명하세요.
3.  B-트리가 데이터베이스에서 사용되는 이유는?

### 힙과 우선순위 큐 ⚙️
1.  최대 힙과 최소 힙의 차이점과 구현 방법은?
2.  힙 정렬의 동작 원리와 시간 복잡도는?

### 해시 테이블 및 기타 비선형 🧩
1.  해시 함수의 조건과 좋은 해시 함수의 특징은?
2.  해시 충돌 해결 방법(체이닝, 오픈 어드레싱)의 장단점은?
3.  HashMap과 TreeMap의 내부 구현 방식과 성능 차이는?
4.  해시 테이블의 시간 복잡도가 $O(1)$인 이유와 최악의 경우는?
