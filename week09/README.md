## ❓ 예상 질문

### 트랜잭션과 무결성
1. **트랜잭션**이란 무엇이며, DB에서 트랜잭션 관리가 왜 중요한가요?
2. 트랜잭션의 **ACID** 속성에 대해 설명하고, 각 속성이 보장하는 역할은 무엇인가요?
3. **격리 수준(Isolation Level)** 4가지(Read Uncommitted, Read Committed, Repeatable Read, Serializable)를 설명하고, 발생 가능한 **Anomaly(이상 현상)**를 비교 설명해 주세요.
4. DB **무결성(Integrity)**의 종류(개체, 참조, 도메인)와 이를 위한 제약 조건은 무엇인가요?
5. **LOCK**과 **Deadlock(교착 상태)**이 발생하는 원인과 해결 방안은 무엇인가요?

### 데이터베이스의 종류
1. **RDB**와 **NoSQL** 데이터베이스의 **가장 큰 차이점**은 무엇인가요?
2. RDB가 **수평 확장(Scale-out)**에 불리한 근본적인 이유는 무엇이며, 이를 극복하기 위한 방법은?
3. 대표적인 NoSQL 모델인 **Key-Value, Document, Column-Family, Graph**의 특징과 적절한 사용 사례를 비교 설명하세요.
4. **CAP 이론**을 설명하고, RDB와 NoSQL이 각각 어떤 속성을 중시하는지 연결하여 설명하세요.
5. **Join 연산**의 원리와 성능에 미치는 영향은 무엇이며, NoSQL 환경에서는 Join을 어떻게 대체하나요?

### 인덱스
1. **인덱스**의 **장점과 단점**을 명확하게 설명해 주세요. 특히 쓰기/수정 성능 저하의 원인은 무엇인가요?
2. **B-Tree**와 **B+Tree**의 **구조적 차이**를 설명하고, DB 인덱스로 주로 **B+Tree**를 사용하는 이유를 설명해 주세요. 
3. **클러스터링 인덱스(Clustering Index)**와 **보조 인덱스(Secondary Index)**의 차이점을 설명하고, 테이블 당 클러스터링 인덱스는 왜 하나만 가능한가요?
4. **복합 인덱스(Composite Index)** 설계 시 **카디널리티**가 높은 순서대로 컬럼을 배열하는 이유를 설명해 주세요.
5. **인덱스 스캔(Index Scan)**, **풀 테이블 스캔(Full Table Scan)**, **인덱스 온리 스캔(Index Only Scan)**의 차이와 각각의 실행 시점은 무엇인가요?
